# Django Blog Application

Djangoフレームワークを使用した、セキュリティとUXを考慮した個人ブログアプリケーションです。
大学2年生として、技術スキルの向上および長期インターン獲得に向けたポートフォリオとして制作しています。

[http://kokiito0906.pythonanywhere.com]
※記事の追加・編集・削除にはログインが必要です。

## ✨ 主な実装機能
チュートリアルをベースに、実用性を高めるための独自カスタマイズを加えています。

### 1. 記事管理（CRUD機能）
- **一覧・詳細表示**: 公開済みの記事をデータベースから取得し、時系列で表示します。
- **新規投稿・編集**: Webフォームを介した直感的な記事操作が可能です。
- **記事削除**: 詳細ページから記事を削除できる機能を実装しました。

### 2. 認証・セキュリティ（認可の制御）
- **ログイン制限**: `@login_required`デコレータを使用し、未ログインユーザーによる投稿・編集・削除を制限しています。
- **UXの工夫**: 未ログイン状態でも「＋（新規投稿）」ボタンを表示させ、クリック時にログイン画面へリダイレクトさせることで、サイトの機能をユーザーに分かりやすく伝えています。
- **CSRF対策**: Django標準のセキュリティ機能を活用し、フォーム送信時の安全性を確保しています。

### 3. 日本語化・地域最適化（L10n/I18n）
- **言語設定**: 管理画面および各UIを日本語に完全対応。
- **タイムゾーン**: `Asia/Tokyo`に設定し、日本時間での正確な投稿管理を実現。

### 4. UI/UXデザイン
- **レスポンシブ対応**: Bootstrap 5を採用し、PC・スマホ両方で快適に閲覧可能です。
- **タイポグラフィの改善**: 日本語の読みやすさを重視し、CSSでフォントファミリーと行間を最適化しました。

## 🛠 使用技術
- **Backend**: Python 3.13 / Django 6.0
- **Frontend**: HTML5, CSS3, Bootstrap 5
- **Database**: SQLite3
- **Infrastructure**: PythonAnywhere (Production), GitHub (CI/CD Workflow)

## 📈 開発プロセス
プロフェッショナルな開発現場を想定したワークフローを採用しています。
- **Git Flow**: 機能ごとにブランチを作成（`improve-login-ux` など）し、動作確認後に `main` ブランチへマージ。
- **デプロイフロー**: ローカル開発環境でのテスト完了後、GitHubを経由して本番サーバー（PythonAnywhere）へデプロイ。
